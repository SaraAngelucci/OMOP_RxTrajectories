Overview

This repository contains a reproducible PySpark pipeline developed for a Master's thesis. The pipeline is drug-agnostic and operates entirely on tables defined by the Observational Medical Outcomes Partnership (OMOP) Common Data Model. It transforms raw, fragmented drug exposure records into:

  1. continuous ingredient-level drug eras;
  2. a monthly person-time grid with active-ingredient burden, starts, stops, and Jaccard turnover;
  3. mutually exclusive monthly prescribing states (e.g. StableMono, StablePolypharmacy, HighTurnover);
  4. four temporally ordered sub-window features that preserve trajectory shape under K-means clustering;
  5. maintenance-aware discontinuation events (early drop-off, restart within 180 days, switch within 60 days);
  6. rule-based discontinuation phenotypes and an unsupervised K-means trajectory clustering, evaluated with a stratified silhouette estimator.
Project pivot. Originally designed to integrate UK Biobank OMOP prescription data with pharmacogenomic variation, access to the secure UK Biobank Research Analysis Platform was unexpectedly revoked late in the timeline. The thesis was refocused on the methodological development and rigorous validation of the underlying phenotyping framework, executed on a Medstat-calibrated synthetic cohort. Because the pipeline reads only OMOP-standard inputs, the same code is portable to UK Biobank, NIH All of Us, and OMOP-mapped Danish National Registers without modification.

1 Install

Python 3.9 or newer and a JVM (for PySpark) are required. Two dependency files are provided: requirements.txt (minimum-floor pins) and requirements.lock (exact pins for bit-for-bit reproducibility).

python -m venv .venv
source .venv/bin/activate
pip install --upgrade pip

# For normal use:
pip install -r requirements.txt

# For exact reproducibility of the thesis numbers:
pip install -r requirements.lock

2 Generate the synthetic cohort

The thesis results are produced on a 1,000-patient synthetic OMOP cohort whose archetype prevalences are mathematically calibrated to Danish primary-care prescribing frequencies derived from the Medstat register (then reproduced with 50,000 patients).

python generate_synthetic_cohort.py --n_patients 1000 --seed 42 \
    --output_dir data/synthetic_medstat

Produces synthetic_drug_era.parquet, synthetic_observation_period.parquet, and synthetic_person.parquet, matching the schemas in src/thesis_rx/io.py.

3 Run the full pipeline

export PYTHONPATH=$(pwd)
python main.py

Runs Experiments 1–3 with config/config_synthetic.yaml.

StepBehaviour
1 · BaselineFull stratified silhouette over k ∈ {2,…,6}; selects best k; writes baseline parquet + silhouette grid CSV.
2 · Negative controlsValidates acute_antibiotic leak rate (synthetic cohort only).
3 · Sensitivity gridARI matrix on discontinuation phenotype labels across a 3×3 (poly × maintenance-day) sweep.

Expect minutes at N≈10³; tens of hours may apply at N≈50,000. If Experiment 3 fails with SparkOutOfMemoryError, increase SPARK_DRIVER_MEMORY.

4 Deterministic logic tests

A separate suite executes the pipeline on a hand-crafted 5-patient cohort encoding every clinically meaningful edge case (stable monotherapy, escalating polypharmacy, intermittent stop-start, acute exposure, right-censoring).

export PYTHONPATH=$(pwd)
python tests/test_validation_cohort.py

Expected: nine primary-cohort logic checks, three edge-case checks, and the message VALIDATION COMPLETE. Pipeline is theoretically sound. Optionally run under pytest.

5 Deploying against real OMOP databases

The pipeline is intentionally decoupled from any specific source dataset; it reads only OMOP-standard tables.

UK Biobank RAP

Point config/config.yaml at the platform's OMOP tables — no code changes required. Handles eid → person_id harmonisation and British date formatting.

Danish Register (LMDB)

For ATC-coded data, danish_register_adapter.py converts LMDB extracts into the same intermediate era schema.

NIH All of Us

OMOP CDM is canonical here. Execute by changing input paths in config/config.yaml; no code changes required.

6 Configuration

All analytic parameters are externalised in YAML and validated by src.thesis_rx.config.validate_config. Key parameters:

KeyDefaultMeaning
analysis.washout_days365Pre-index observable time required for eligibility
analysis.followup_months24Length of the participant-specific monthly grid
analysis.exposure_gap_days30Era-merge gap for the sensitivity analysis
analysis.early_discontinuation_days90Max duration of a first era classified as early drop-off
analysis.restart_window_days180Window for restart events
analysis.switch_window_days60Window for switch events
analysis.polypharmacy_threshold5Concurrent ingredients defining polypharmacy
analysis.turnover_low / high0.20 / 0.50Jaccard cut-points for state classification
clustering.k_grid[2,3,4,5,6]Candidate k for K-means
clustering.seed42Random seed

7 Reproducibility checklist

  • All parameters externalised in config/*.yaml.
  • All random seeds fixed (numpy, Python random, Spark KMeans seed).
  • All intermediate outputs persisted as Parquet under cfg['project']['output_dir'].
  • Deterministic unit-test cohort reproducible bit-for-bit.
  • The Medstat-calibrated synthetic generator is fully deterministic given a seed.
